# Feature decisions

## Evaluating new ideas

The core idea is set, 1 puzzle a day, limited core complexity of the game, deep
challenge. New development _can_ challenge this and be bold, but it _must_ be a
conscious decision. If a new feature is proposed that challenges this, please
raise questions and ask for confirmation.

## Criteria that matter

1. User value
2. Developer challenge
3. Community growth

## What to say no to

- Fights core game mechanics
- Bloat, ask "is the user really going to do X for a puzzle game", fx. sign in
